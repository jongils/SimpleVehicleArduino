시스템 구성

조이스틱 입력부터 CAN 전송까지, 5개 레이어로 구성된 실시간 파이프라인

🕹️

① 입력 계층

YwRobot JoyStick Shield로 운전자 명령을 수집합니다.

  • A0 → 조향각 (±180°)
  • A1 → 가속·제동 [0–1]
  • D6 → 푸시 버튼
  • 데드존(±50 ADC) 처리 내장
⚙️

② 엔진·드라이브트레인

포물선형 토크 곡선으로 엔진 출력을 모델링하고 후륜 오픈 디퍼런셜로 구동력을 분배합니다.

  • 최대 RPM: 6,000
  • 피크 토크: 200 Nm @ 3,500 RPM
  • 기어비: 0.95, 파이널 드라이브: 3.73
🔬

③ 물리 적분 엔진

RK4 또는 Euler로 매 루프마다 차량 상태를 업데이트합니다.

  • 종방향 1-DOF (속도·가속도)
  • 횡방향 2-DOF (자전거 모델)
  • 타이어 슬립·횡력 계산
  • 4-휠 법선력 (무게 이동 반영)
📡

④ CAN 전송

MCP2515 컨트롤러를 통해 500 kbps로 8개 프레임을 100 Hz 주기로 전송합니다.

  • ID 0x100 — 엔진 데이터
  • ID 0x101~0x104 — 4-휠 데이터
  • ID 0x200 — 조향 데이터
  • ID 0x201 — 횡방향 동역학
  • ID 0x300 — 차량 속도·가속도
🖥️

⑤ 직렬 대시보드

ANSI 이스케이프 코드로 2 Hz 실시간 터미널 대시보드를 출력합니다.

  • 115,200 baud 시리얼
  • 입력·조향·엔진 상태
  • 4-휠 슬립·타이어력
  • 종·횡 동역학 진단

VehicleDynamicsCAN.ino — 877줄의 핵심 Arduino 스케치

핵심 함수 목록

각 함수는 단일 책임 원칙을 따르며, 물리 계층별로 명확히 분리되어 있습니다.

readJoystick() ADC 입력을 스로틀·제동·조향각으로 매핑. 데드존 및 포화 처리.
engineTorque() 현재 RPM 기반 포물선 토크 곡선 계산. 아이들·레드라인 경계 처리.
updateAckermann() Ackermann 기하학으로 4-휠 조향각 계산. 전방 독립 조향.
normalForces() 종방향 무게 이동을 반영한 4-휠 법선력 계산.
integrateWheel() 휠 각속도 Euler 적분. 토크 → 각가속도 → 각속도.
updateDynamics() 메인 물리 적분기. RK4 또는 Euler 선택 (#define).
latEval() RK4 중간 단계용 횡방향 미분 평가 (자전거 모델).
lonDeriv() RK4 중간 단계용 종방향 가속도 계산 (공기저항 포함).
sendCANFrames() 차량 상태를 8개 CAN 프레임으로 인코딩 후 전송.
printDashboard() ANSI 포맷으로 실시간 상태를 시리얼 터미널에 출력.
VehicleDynamicsCAN.ino — 컴파일 타임 옵션 
// ── 수치 적분법 선택 ──────────────────────────
#define USE_RK4   1  // 1=RK4(정밀), 0=Euler(경량)

// ── 차량 파라미터 (조정 가능) ─────────────────
const float MASS        = 1500.0f;   // [kg]
const float PEAK_TORQUE  = 200.0f;   // [Nm]
const float MAX_RPM     = 6000.0f;
const float WHEELBASE   = 2.70f;    // [m]
const float TIRE_RADIUS = 0.315f;   // [m]
const float MU         = 0.90f;    // 마찰계수
const float CD         = 0.20f;    // 공기저항 계수
메인 루프 구조 
void loop() {
  // ① 자유 주기 물리 적분 (micros 기반 가변 dt)
  float dt = (micros() - lastTime) * 1e-6f;
  readJoystick();
  updateDynamics(dt);      // RK4: 4회 미분 평가

  // ② CAN 전송 — 100 Hz (10 ms마다)
  if (millis() - lastCanTime >= 10) {
    sendCANFrames();
    lastCanTime = millis();
  }

  // ③ 시리얼 대시보드 — 2 Hz (500 ms마다)
  if (millis() - lastSerialTime >= 500) {
    printDashboard();
    lastSerialTime = millis();
  }
}
엔진 토크 곡선 
float engineTorque(float rpm, float throttle) {
  // 포물선형 토크 곡선: 아이들~레드라인
  float r = (rpm - IDLE_RPM) / (MAX_RPM - IDLE_RPM);
  r = constrain(r, 0.0f, 1.0f);
  // 3,500 RPM에서 피크, 양 끝에서 0으로 수렴
  float curve = 4.0f * r * (1.0f - r);
  return PEAK_TORQUE * curve * throttle;
}
Physics Models

물리 모델 상세

AVR 8비트 MCU에서 구현된 4가지 핵심 역학 모델

종방향 1-DOF

종방향 동역학

타이어 구동력에서 공기저항·구름저항을 차감한 합력으로 차체 가속도를 계산합니다. 저속(6 km/h 미만) 에서는 지수 감쇠로 부드럽게 정지합니다.

ax = (ΣFtire − Fdrag − Froll) / m
Fdrag = 0.5 × ρ × Cd × A × vx²
vx(t+dt) = vx(t) + ax·dt
횡방향 2-DOF

자전거 모델 (Bicycle Model)

전·후 등가 축으로 단순화한 선형 자전거 모델로 횡속도와 요레이트를 동시에 적분합니다.

dVy/dt = (Fyf + Fyr)/m − r·vx
dr/dt = (lf·Fyf − lr·Fyr) / Iz
α_f = δ − atan((Vy + lf·r)/vx)
타이어 모델

선형 슬립 모델 (μFz 포화)

종방향 슬립비와 슬립 강성으로 타이어 구동·제동력을 계산하고, μ×Fz로 포화시킵니다. 법선력은 종방향 무게 이동을 반영합니다.

κ = (v_wheel − v_body) / max(|v_w|,|v_b|)
Fx = Cs × κ, |Fx| ≤ μ·Fz
Fy = Cf × α, |Fy| ≤ μ·Fz_axle
수치 적분

RK4 / Euler 이중 지원

#define USE_RK4로 컴파일 타임에 선택합니다. RK4는 4회 미분 평가로 O(dt⁴) 오차를 달성하며, Euler는 AVR 자원이 부족할 때 대안으로 사용합니다.

RK4: k1=f(t,y), k2=f(t+h/2, y+k1·h/2)
k3=f(t+h/2, y+k2·h/2), k4=f(t+h, y+k3·h)
y(t+h) = y + (k1+2k2+2k3+k4)·h/6
